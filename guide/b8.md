# Lịch thi học kỳ (Chống RTE PTIT)

## Đề bài
Cho danh sách N ca thi gồm Ngày thi, Giờ thi, Phòng thi. Sắp xếp các ca thi theo thời gian tăng dần.

## Ý tưởng cốt lõi (Tuyệt kỹ chống RTE)
1. **RTE (Run Time Error) là gì?** 
   Khi PTIT chấm bài, họ hay chèn thêm các dòng trống (enter thừa) hoặc dấu cách vô nghĩa ở cuối file test. Lệnh `input()` bình thường sẽ đọc phải dòng trống, dẫn đến gãy chương trình khi đang cố cắt chuỗi (ví dụ `date.split('/')`).
2. **Cỗ máy nghiền Input bất tử**:
   Sử dụng thư viện `sys`. Lệnh `sys.stdin.read().splitlines()` sẽ hút sạch sành sanh toàn bộ văn bản trong file test.
   Sau đó dùng `[l.strip() for l in raw_data if l.strip()]` để:
   - `strip()`: Gọt sạch khoảng trắng thừa ở hai đầu.
   - `if l.strip()`: Lọc bỏ hoàn toàn các dòng trống, chỉ giữ lại những dòng có chữ.
   Nhờ mưu kế này, PTIT có chèn thêm 1 tỷ dòng trống code ta vẫn sống nhăn răng!
3. **Sắp xếp thời gian (Tuple comparison)**:
   Để sắp xếp nhanh, ta ép ngày, tháng, năm, giờ, phút thành một Tuple `(năm, tháng, ngày, giờ, phút)`. Python sẽ tự động so sánh Tuple từ trái qua phải (Năm trước, Tháng sau...). Vô cùng nhàn nhã!

## Pseudocode
```text
Khuôn Exam:
    Khởi tạo (id, ngày, giờ, phòng):
        Mã = "C" + id (3 chữ số)

        Tách giờ/phút từ chuỗi giờ
        
        Key_sắp_xếp = (năm, tháng, ngày, giờ, phút)
        
    Luật so sánh (<):
        So sánh Key_sắp_xếp (thời gian bé hơn đứng trước)
        Nếu bằng nhau thì so Mã (Mã bé hơn đứng trước)

Chương trình chính:
    Đọc TOÀN BỘ file bằng sys.stdin
    Lọc bỏ TẤT CẢ các dòng trống
    
    N = Dòng đầu tiên
    Lặp N lần (đọc nhảy mỗi bước 3 dòng):
        Ngày = dòng[idx]
        Giờ = dòng[idx+1]
        Phòng = dòng[idx+2]
        Tạo Exam và ném vào mảng
        
    Mảng.Sắp_xếp()
    In ra danh sách
```
