# PY02028 - Sắp xếp nguyên tố

## Đề bài
Cho dãy số `A` có `N` phần tử. Yêu cầu giữ nguyên vị trí các số KHÔNG PHẢI là số nguyên tố. Đối với các số nguyên tố, hãy sắp xếp chúng theo thứ tự tăng dần và đặt vào đúng các vị trí ban đầu của nó.

## Ý tưởng cốt lõi
Bài này dùng chiến thuật cực kỳ an toàn và mạch lạc: **"Rút ruột - Nắn xương - Bơm lại" (Extract - Sort - Inject)**.
1. **Rút ruột:** Duyệt mảng ban đầu, nếu là số nguyên tố thì nhặt riêng ra một mảng `b`.
2. **Nắn xương:** Đem mảng `b` đi sắp xếp tăng dần bằng `b.sort()`.
3. **Bơm lại:** Duyệt mảng ban đầu thêm một lần nữa. Cứ thấy phần tử nào là số nguyên tố thì lôi phần tử tương ứng trong mảng `b` ra nhét đè lên. Dùng một biến `j` để đếm xem đã bơm đến phần tử thứ mấy của mảng `b`.

## Pseudocode

### Luồng xử lý chính
```text
Bắt đầu:
    Đọc số N và mảng A
    Khởi tạo mảng rỗng B
    
    // Bước 1: Rút ruột
    Lặp i từ đầu đến cuối mảng A:
        Nếu A[i] là số nguyên tố:
            Nhét A[i] vào mảng B
            
    // Bước 2: Nắn xương
    Sắp xếp mảng B tăng dần
    
    // Bước 3: Bơm lại
    Khởi tạo j = 0 (Con trỏ của mảng B)
    Lặp i từ đầu đến cuối mảng A:
        Nếu A[i] là số nguyên tố:
            A[i] = B[j]  // Đè số nguyên tố đã sắp xếp vào đúng ổ
            j = j + 1    // Tiến con trỏ mảng B lên 1 bước
            
    In mảng A (nhớ dùng print(*a) để in cách nhau bởi dấu cách)
```

## Pythonic Way (Phương pháp tối ưu)
Chính cách viết của Chủ nhân đã là TỐI ƯU cho tư duy phòng thi rồi! Dễ nhớ, dễ debug, chạy cực mượt.

```python
import math

# Hàm kiểm tra nguyên tố thần tốc
def checksonguyento(n):
    if n < 2: return False
    for i in range(2, math.isqrt(n) + 1):
        if n % i == 0: return False
    return True

n = int(input())
a = list(map(int, input().split()))

# 1. Rút ruột
b = []
for i in range(len(a)):
    if checksonguyento(a[i]):
        b.append(a[i])

# 2. Nắn xương
b.sort()

# 3. Bơm lại
j = 0
for i in range(len(a)):
    if checksonguyento(a[i]):
        a[i] = b[j]
        j += 1

print(*a)
```

**Bài học rút ra:** 
Không cần phải cố viết code 1 dòng (List Comprehension/Iterator) nếu nó khiến não bị quá tải trong phòng thi. Việc tách bạch thành các vòng `for` đơn giản, dễ nhớ như cách làm này đôi khi lại là vũ khí sắc bén và an toàn nhất!
