# Tính tiền nước (OOP - Giá bậc thang)

## Đề bài
Tính tiền nước cho N khách hàng dựa vào chỉ số cũ và mới. Đơn giá được tính theo bậc thang:
- $\le 50$ m3: giá 100, phụ phí 2%
- $51 - 100$ m3: giá 150, phụ phí 3%
- $> 100$ m3: giá 200, phụ phí 5%
Sắp xếp hóa đơn theo số tiền giảm dần.

## Ý tưởng cốt lõi
1. Bài này tương tự bài Bảng điểm, ta dùng **Class Customer** để lưu thông tin.
2. Dùng cấu trúc `if-elif-else` để tính giá trị `usage = new - old`.
   - **Chú ý tính bậc thang:** Nếu xài 70 khối, ta KHÔNG lấy $70 \times 150$. Ta phải lấy 50 khối đầu $\times 100$, cộng với 20 khối dư $\times 150$.
   - **Phụ phí:** Tổng tiền = Tiền gốc x (1 + phần_trăm). Ví dụ phụ phí 2% tức là x 1.02.
3. Làm tròn thành số nguyên bằng hàm `round()`.
4. Ghi đè hàm `__lt__` (Less Than) để giúp lệnh `sort()` sắp xếp tự động từ lớn đến bé theo tổng tiền.

## Pseudocode
```text
Tạo Khuôn Customer:
    Khởi tạo (id, tên, số_cũ, số_mới):
        Mã = "KH" + id (định dạng 02 số)
        Tên = tên
        
        Số_m3 = số_mới - số_cũ
        
        Nếu Số_m3 <= 50:
            Tiền_gốc = Số_m3 * 100
            Tổng_tiền = Tiền_gốc * 1.02
        Ngược lại Nếu Số_m3 <= 100:
            Tiền_gốc = 50*100 + (Số_m3 - 50)*150
            Tổng_tiền = Tiền_gốc * 1.03
        Ngược lại:
            Tiền_gốc = 50*100 + 50*150 + (Số_m3 - 100)*200
            Tổng_tiền = Tiền_gốc * 1.05
            
        Làm_tròn(Tổng_tiền) thành số nguyên
        
    Luật Sắp Xếp:
        So sánh Tổng_tiền, ai lớn hơn đứng trước. Bằng nhau thì so Mã.

Chương trình chính:
    Nhập N khách hàng
    Cho vào danh sách Customers
    Customers.Sắp_xếp()
    In ra Mã, Tên, Tổng_tiền
```
